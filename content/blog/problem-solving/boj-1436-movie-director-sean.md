## 문제

BOJ 1436번 : [영화감독 숌](https://www.acmicpc.net/problem/1436)

## 접근 방법

*666*이 연속으로 있는 숫자들을 찾으면 된다.

### 방법 1.

내가 사용한 방법이다. 3-7자리까지 나올 수 있는 경우의 수를 계산해봤는데 <u>약 3,000,000까지의 범위 내에 대략 10,000개의 경우의 수</u>가 나온다. 입력 범위가 10,000까지 이므로 3,000,000 내 *666*이 연속으로 있는 숫자들을 다 추출하면 된다. 이 때 `lambda`식과 `filter`함수를 사용하면 한 줄로 깔끔하게 추출할 수 있다.

마지막으로 인덱싱으로 `N-1` 번째 숫자를 뽑으면 된다.

### 방법 2.

첫 번째 방식은 내가 머리쓰는 시간을 생각하면 그리 효율적이지는 않은 것 같다ㅋㅋ 경우의 수를 일일이 구해서 대략적인 범위를 계산해야하니까. 다른 분들의 풀이를 찾아보니 [bws96o님의 풀이](https://www.acmicpc.net/source/21902130)가 가장 깔끔해 보여서 정리해보았다.

- *666*이 포함된 숫자를 세는 `cnt` 그리고 1씩 계속해서 증가시키는 `num`을 선언한다.
- `cnt`가 `N`보다 작을 동안 `num`을 1씩 증가시킨다.
- 만약 `num`에 숫자 *666*이 포함되어 있다면 `cnt`를 1 증가시킨다.
- 루프를 다돌고 `num`을 출력한다.

<br>

개인적으로 num을 *666*부터 시작하면 조오금 더 빠르지 않을까 생각이든다. 참고로 **방법1**은 _856ms_, **방법2**는 _932ms_, **666부터 세기 시작한 방법**은 *912ms*가 걸렸다.

## 교훈

> 생각하는 시간 vs 실제 프로그램 실행 시간

어느 게 더 중요할까 좀 고민을 해봐야겠다. 사실 효율성을 따지자면 후자가 맞는데 코딩테스트나 알고리즘 대회라면 전자가 더 짧은 게 좋지 않을까.

## 소스 코드

### 방법 1.

```python
N = int(input())
nums = list(filter(lambda x: '666' in str(x), range(666, 3000000)))
print(nums[N-1])

```

### 방법 2.

```python
# 참고 :  https://www.acmicpc.net/source/21902130
N = int(input())
cnt, num = 0, 665
while cnt < N:
    num += 1
    if '666' in str(num):
        cnt += 1

print(num)
```
