## 최적화

gereralization
training error와 test error의 차이를 말한다. 그래서 학습 데이터 성능이 안 좋아도 generalization이 좋을 수도 있다. 예를 들면, 에러가 50%가까이 되는데 training과 test error의 차이가 1%도 안 나는 경우..

underfitting vs overfitting
언더피팅 - 모델이 덜 학습된 것
오버피팅 - 모델이 학습데이터에만 너무 맞게 학습된 것
둘 다 안 좋고 적절히 밸런스 가진 게 좋음. 근데 이상향..

cross validation
모델 검증 테크닉 - kfold 생각 하면 될 듯
보통 kfold라고 많이 하고
최적 하이퍼파라미터를 찾을 때 사용하고 a 넣어보고 돌려보고 b 넣어보고 돌려보고 해서 더 좋은 거 고르고

다만 test data는 절대 쓰면 안 됨 흔히 답 알고 수능친다 이런 느낌

bias-variance trade-off

variance - 데이터를 넣었을 때 출력이 얼마나 일관하게 나오는지
bias - 퍼져 있어도 평균적으로 봤을 때 출력이 true target에 가깝다면 bias가 낫다고 한다

근데 이게 왜 중요? 우리가 cost를 분해하면 bias + variance + noise로 분해할 수 있음 그래서 둘 다 줄이긴 힘들고 적절히 맞춰서

bootstrapping

학습 데이터를 일부 샘플링해 모델을 만들고 그걸 반복하는 것. 모델의 예측값을 비고해서 전체적인 모델의 불확실성을 예측하고자 할 떄 사용한다.

결과를 뽑는 방법으로는 voting과 averaging이 있음

boosting

## 규제 방법

- early stopping: validation error와 training error의 차이가 커지는 시점에 학습을 멈춤 이 경우 추가로 검증 데이터가 필요
- parameter norm penalty = weight decay :functin space에 부드러움을 추가? weight decay라고 걍 패널티를 주는 거임 보통 cost값 구할 때 손실함수값만 가지고 구하는데 거기다가 가중치의 l2 norm값을 둬서 가중치가 클수록 패널티가 증가하는데 즉, 큰값을 더해 손실함수 값을 더 키움
- data augmentation: 데이터를 요리 조리 변형하는 것 근데 라벨이 바뀌지 않는 선에서 라이언 귀엽다
- noise robustness 랜덤 노이즈 추가하는 건데 input이나 weight에 교수님꼐서 음.. 왜 그런지는 아직 모른다고 함
- label smoothing decision boudary를 부드럽게 마드는 것 - 2개의 데이터를 랜덤으로 뽑아서 입력과 출력을 섞는 것 -> 사진 mixup 성능 좋다고 함
- dropout: 노드 몇개 0으로 만들어서 떨구는 것 robust하게
- batch normalization 논란이 있긴해! 활성화 함수 전에 선형결합 결과를 정규화시키는 것 근데 깊게 쌓을 때 이거 넣으면 꽤 성능이 나온다네? 여러 개의 변형이 있음
