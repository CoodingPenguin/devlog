## Generative Model

생성 모델은 GAN, VAE같은 그럴싸한 것을 만드는 것 외에 무궁무진한 다른 무언 가도 포함하는 큰 집합

강아지 이미지가 있다고 가정해보자.
생성모델은 다음을 한다.

1. 이미지 생성: 학습 데이터에 있지 않은 강아지 이미지를 만들어냄 (강아지 이미지의 확률분포에 근사한 어떤 확률분포에서 샘플링했을 때 그게 강아지 이미지랑 비슷해)
2. 밀도 측정: p(x)가 높으면 강아지일 확률니 높고 아니면 강아지가 아닐 확률이 낮음 -> 분류기 이상행동감지도 가능 -> 단순 생성만 하는게 아니므로 explicit model
3. 비지도 표현 학습: 강아지의 특징을 학습함

그럼 확률분포 즉, x를 샘플링할 수 있는 p(x)는 어떻게 나타낼까?

우리는 이산확률분포 finite set으로 표현가능한 분포가 필요해

- 베르누이 분포 -> p(x)를 표현하려면 변수 1개만 필요 / p, 1-p로 표현 가능하니까
- 카테고리 분포 -> p(x)를 표현하려면 변수 n-1개만 필요 (카테고리가 n개라고 할 때) / 왜? p_1, p_2, ..., p_n-1, 마지막 p_n = 1 - p_1 + p_2 + ... p_n-1로 표현 가능

예시. rgb 결합 분포
(r, g, b) ~ p(R, G, B)
R, G, B가 서로 독립일 때
1 픽셀이 가질 수 있는 컬러의 수는 256 _ 256 _ 256이야.
그럼 저 컬러의 수를 표현하기 위한 변수의 개수는?
다 독립이고 각 R, G, B는 카테고리 분포를 띄니까
255 *255 *255가 된다.
여기서 중요한 것은 1개의 픽셀의 컬러를 표현하기 위한 파라미터(=변수)가 많이 필요하다는 것

예시. BINARY IMAGE
X_1, ... , X_n n개의 픽셀이 있음
이미지를 표현할 때 표현가능한 경우의 수는? 각 픽셀마다 0 또는 1이므로 2^n이 되지
그럼 그 경우의 수를 표현하기 위한 변수의 개수는?
p(x_1, x_2, ... x_n)를 표현하기 위한 파라미터의 개수는?
일단 각 X_1은 베르누이 분포를 따르고 + dependent하다고 생각하자.
이거는 뒤에서 다시 설명 (chain rule 써서 하는 게 더 훨씬 나을 듯)
2^n - 1 왜? 마지막 경우의 수는 하나 빼면 되니까..?

X_1, ..., X_n이 독립이라고 생각해보자.
p(x_1, ... , x_n) = p(x1)p(x2) ... p(xn)이 되고
각 확률분포는 베르누이 확률분포를 따르므로 각 변수는 1개만 필요하고
결국 2^n의 경우의 수를 표현하는데 필요한 변수는 n개가 된다.
독립을 가정하면 파라미터의 수가 많이 줄어든다!

근데 fully depdent하면 파라미터수가 너무 많고
fully indepdent하면 너무 말이 안되니까
이 중간 지점을 찾아보고자해

조건부 독립

중요한 컨셉
chain rule
얘는 확률변수가 독립이든 의존이든 상관없이 성립하는 수식

베이즈 정리 뭐 이런게 있다! 이거도 똑같음

조건부 독립은 가정인데
이거 그 뭐냐 이거 아래거 참고
https://actruce.com/conditional-independence/
무슨 말이냐면 z가 주어질 때 x와 y가 독립이라 가정하면
y, z가 주어질 때의 x의 확률분포는 z가 주어질 때 x의 확률분포와 같다.
여기서 중요한 건 y가 날라간다는 것!

그래서 앞에서 했던 걸
chain rule으로 나타내면
결국 2^n-1의 변수가 필요한데
중요한 건 아직 가정을 하지 않음! 아직 fully dependent한 상태임

이제 markov 가정을 해
이건 직전 과거 정보에만 dependent하다는 가졍이야
이걸 쓰면 파라미터 수가 2\*n-1로 줄어든다.

이런 조건부 독립을 취한 모델이 바로 autoregressive model이야

## Latent Variable Model

오토인코더가 생성모델인가? 아님!
생성하는 건 아니고 그럼 vae가 어떻게 생성모델이 되었는지를 알아보는 게 목표

vae

vi

- 사후확률분포를 찾는 것이 목표 - 사후확률분포는 내가 관찰한 사항이 주어질 때 관심있는 것에 대한 확률분포
  vd 이 사후확률을 근사한 확률분포
  우리는 사후확률분포를 계산하기 힘듦으로 이를 근사하는 데 이 과정을 vi라고 한다.

그럼 우리는 사후확률분포를 찾으려고하는데 여기서 중요한 건 목적함수를 설정하는 것
그래서 우리는 목적함수를 kl-divergence pd와 vi사이의

이것이 encdoer가 하는 일

근데 우리가 사후확률분포도 모르는데 이걸 어떻게 근사해?
kl을 줄이는 것이 목적인데

데이터의 확률분포를 전개해봤는데
elbo와 objective로 나뉘고
우리는 objective를 줄이고자 하는 게 목적인데 이게 불가능하니 (target을 모르니까)
반대급부로 elbo를 최대화하고자함
어렵게 생각할 것이 아니라 objective를 줄이는 것이 elbo를 최대화하는 것과 똑같은 거야
그래서 elbo를 최대화해서 objective를 줄이는 거지

다시 정리하면
x를 잘 표현하는 z라는 잠재공간을 만들고 싶어
z라는 잠재공간 즉, p(x|z)를 모르니까 이걸 찾기 위해 encoder로 근사를 하고 싶어.
근데 p(x|z)를 모르니까 근사를 하룻 없는데
vi가 elbo를 최대화해서 kl을 줄여주는 효과가 있으니까 elbo를 최대화시켜려고 하는거야

근데 elbo를 또 전개해서 보니
reconstruction term과 prior fitting term으로 나뉘어져
이걸 구현한 것이 vae (implicit)

하지만 여기서 중요한게
이건 좀 힘든게 prior fitting term이 미분이 가능해야 하는데.. 왜냐하면 elbo를 최대화시킬 거니까?
근데 이게 적분도 있고 해서 closed form이 잘 안 나와..
그래서 어떻게 하면 isotropic gaussian을 따른다고 가정(prior distribution이)을 하고
이제 전개를 하면 이제 좀 closed form이 나오게 돼.
그럼 훨씬 깨긋해지니까 학습하기 좋겠지?

그럼 이제 생성은 어떻게 할까?
latent space의 prior distribution에서 z를 샘플링해서 걔를 디코더에 넣어서 생성하는 것

결과는 이렇다..!

adversial auto-encoder
vae의 큰 단점 : 인코더에서 prior fitting term이 kl-divergence를 사용한다는 것 그래서 가우시안 분포를 가정해야하고 만약 아닌 경우에는 활용을 할 수가 없어

그래서 adveris auto-encoder는 gan을 활용해서 latent distribution 사이에 분포를 맞춰준다.
prior fitting term을 gan object로 바꾼다.
latent dist을 샘플링 가능한 ㅇ떤 분포만 있어도 이것과 맞춰?
어쨌든 다양한 분포 활용이 가능하다!
성능도 좋아!
