## Pythonic Code

- 리스트 컨프리헨션에서 조건으로 3항 연산자를 쓴다면 이때는 조건문이 뒤가 아니라 앞으로 온다
- pprint 쓰면 더 예쁘게 출력 가능
- 리스트 컨프리헨션에서 2개의 for문을 쓰는 것은 nested loop을 쓰는 것과 같다. zip과 헷갈리지 말 것.
- map의 경우 리스트 2개 적용이 가능함. 즉, 함수의 인자로..
- 근데 map, reduce, lambda 다 PEP8에서 쓰지 않을 것을 권장.
  - 왜? 가독성 너무 떨어짐..
  - 하지만 아직 많이 쓰인다. 특히 reduce는 대용량 데이터 처리에 자주 쓰임.
- iterable - 열혈 파이썬이랑 처음 시작하는 파이썬 다시 공부
- generator도 동일
  - 바로 생성해서 메모리에 올린다 = iterable
  - 주소값만 갖고 호출 할 때 마다 값을 생성해서 하나씩 메모리에 올린다 = generator
  - 대용량 데이터 처리에 좋다. 메모리에 많은 것을 올리지 않으니까
  - 나중에 정말 많이 쓰인다고 한다.
- 함수 인자 순서: 일반 인자 - 가변 인자 - 키워드 가변 인자
  - `param, *args, **kwargs` 순이다! 이거 중요하다.
  - 즉, 가변인자가 일반 인자보다 앞으로, 키워드 가변인자가 가변 인자보다 앞으로 올 수 없다.
- 그 외에 \*와 \*\*은 언패킹 하는데도 사용된다.
  - 즉, 약간 차원을 하나 낮춘다는 느낌? 그런 느낌이 크다.
    - 예를 들면, `[1, 2, 3, 4, 5]` 이런 리스트가 있으면 그걸 벗기고 1, 2, 3, 4, 5로 푸는 그런 느낌
  - 근데 어디서 언패킹? 사실 어디서도 써도 가능한데 함수의 인자로 넣을 떄? 그 때 자주 사용한다.
  - 굳이 풀어서 쓰지 않고 한방에! 약간 자바스크립트의 `...`를 생각하면 쉽다.
