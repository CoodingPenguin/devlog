## Variable & List

변수

변수는 메모리 주소를 가지고 있고 변수에 들어가는 값(value)은 메모리 주소에 할당된다.


기본 자료형 primitive data types
파이썬이 처리할 수 있는 데이터 유형으로 각 유형별 메모리에 차지하는 공간 크기가 다르다.
표

자료형
그럼 변수의 데이터 타입은 언제 결정?
다이나믹 타이핑이라고 했잖아!
값이 할당되면서 그 값에 따라 변수의 자료형이 결정된다.

자료형에 따라 할당되는 크기가 달라 표현 범위도 다르다.


리스트

시퀀스 자료형으로 여러 데이터들의 집합
다른 프로그래밍 언어와는 달리 요소의 자료형이 모두 동일하지 않아도 된다.

인덱싱: 주소(offset)을 통해 해당 주소에 있는 값을 호출한다.
슬라이싱: 주소를 기반으로 부분 값들을 반환한다.
- offset 범위가 넘어가도 ok
- step을 이용해서 역정렬도 가능하다.
  
리스트 함수 -> 주소 남겨놔

리스트의 복사
변수=변수 -> 같은 값을 가리키게 된다. sysgetref이런걸로 찍어보자.
변수=변수[:] -> 얕은 복사 안에 있는 요소 모두가 primitive type이면 상관없다. 하지만, 참조형이라면..
변수=copy.deepcopy 어쩌구 해야한다. 왜? 리스트의 요소가 참조형이면 그 요소도 해댱 객체를 가리키는 상태이기 때문에
복사가 되도 주소 복사가 일어나기 떄문이다. 이차원 리스트 같은 거는 무조건 deepcopy 써야함.

패킹, 언패킹 -> 튜플에서도 동일하게 쓰인다.

## String and Advanced Function

문자열
시퀀스 자료형으로 문자열 데이터를 메모리에 저장한다.
영문자 한 글자는 1 Byte의 메모리 공간을 사용한다.

문자열 리스트처럼 인덱싱, 슬라이싱 가능
문자열 함수 -> 링크

특수 문자 -> 표

raw string
특수 문자의 특수 기호인 \을 무시하고 그대로 출력

함수 호출 반식
call by value
함수에 인자를 넘길 떄 값만 넘긴다. 
call by reference
함수에 인자를 넘길 때 메모리 주소를 넘긴다.
call by object reference
객체의 주소가 함수로 전달된다.
원본을 지키기 위해서는 복사해줘야 한다.(습관)

변수의 범위
변수가 사용되는 범위
지역 변수: 함수내
전역 변수 : 프로그램 전체
- 함수 내에서 사용 가능 하지만 함수 내 전역변수와 이름이 같은 변수가 생기면 지역 변수가 우선
- 전역 변수임을 명시해주기 위해서 global 키워드 사용
