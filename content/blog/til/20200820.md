## ✨ 오늘 한 일

- **백준 브루트포스 [체스판 다시 칠하기](https://www.acmicpc.net/problem/1018) 문제를 풀었다.**
  - 이 문제는 **두 가지 방식**으로 풀 수 있다. 하나는 <u>체스판 마스크를 만들어 덮어 씌워 다른 부분의 개수를 세는 방식</u>이다. 근데 8x8로 2개를 만드는 게 귀찮아서 관뒀다ㅋㅋ 다른 하나는 <u>8x8의 window가 있다고 생각하고 움직이면서 개수를 세는 것</u>이다.
  - 나는 후자로 <u>[repl.it](repl.it)</u>에서 풀었다. 근데 아무리 봐도 틀린게 없는데 계속 `런타임 에러`가 나서 무려 한 시간을 해맸다. 이유는 **백준에서는 numpy나 pandas와 같은 라이브러리를 지원하지 않아서**였다😭 내 풀이는 numpy가 필요했는데 numpy를 백준에서 지원을 안 해주니 계속 에러가 났던 것이다.
- **[Start Here: A Gentle Introduction - Will Koehrsen](https://www.kaggle.com/willkoehrsen/start-here-a-gentle-introduction) 1차 필사를 드디어 완료했다!**
  - 오늘은 *Baseline*을 만들었다! *Baseline*이라는 용어가 참 생소했는데 구글링 해보니 **모델 설계시 기준으로 삼을 모델**을 말한다. 보통 머신러닝 모델을 만들게 되면 개선되거나 복잡한 모델을 만드는데 그 전에 정말 간단한 나이브-베이즈 모델이나 로지스틱 회귀 모델처럼 간단한 모델을 가지고 `기준선`을 만드는 것이다.
  - 이렇게 기준 모델을 만들어두면 복잡한 모델을 만들고 성능 평가를 할 때 성능이 잘 나왔는지 파악할 수 있는 지표가 될 수 있다.
- **[파이썬 웹 프로그래밍](http://www.yes24.com/Product/Goods/64100462) 2.2장. 웹 클라이언트 라이브러리까지 공부했다.**
  - `urllib` 모듈과 `http.client`를 사용해서 간단한 **웹 클라이언트 프로그램**을 만들어보았다! 책 설명이 자세해서 그렇게 어렵지는 않았다. <u>연결객체생성 → 요청 → 응답객체생성 → 응답메시지받음 → 연결해제</u> 순인데 소캣 프로그래밍의 순서랑 비슷해서 금방 이해할 수 있었다.
  - *GET*말고 *POST*나 *PUT 메소드*로 HTTP 통신을 할 때 서버가 필요한데, 책에 제공된 코드로 Django 서버를 로컬에서 돌렸다. 근데 왠지는 모르겠지만은 통신은 되는 것 같은데 계속 에러 메시지가 뜬당ㅠㅠ 내일 한 번 원인을 찾아봐야겠다.
